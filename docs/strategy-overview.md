# 策略总览

本文档梳理当前仓库中已经存在的交易策略与选股策略，区分哪些已经接入模拟/回测，哪些仍然是信号输出或决策框架，便于后续做实验、回测或参数研究。

## 总表

| 策略 | 类型 | 是否可直接回测/模拟 | 关键参数/核心条件 | 更适合的市场环境 |
|---|---|---|---|---|
| `buy_and_hold` | 交易策略 | 是 | 基本无参数，买入后持有 | 用作基准，对照主动策略是否有超额收益 |
| `sma_cross` | 交易策略 | 是 | `fast`、`slow`，快线上穿慢线买入、下穿卖出 | 趋势明显、波段清晰的环境 |
| `rsi_revert` | 交易策略 | 是 | `buy_rsi`、`sell_rsi`，超卖买入、超买卖出 | 震荡或均值回归特征更强的环境 |
| `a-share-strategy-mainboard-multi-swing-defensive` | 选股 + 买卖信号 | 否，需自行接模拟盘或回测层 | 主板流动性池、`trend_pullback`、成本过滤、鲁棒性过滤 | 主板趋势回踩、偏防御型波段环境 |
| `macd-second-golden-cross` | 单票交易决策 | 否 | 前期下跌、第一脚/第二脚、零轴下二次金叉、价格确认 | 低位修复、超跌反弹、筑底阶段 |
| `macd-trend-resonance-stock-picker` | 选股策略 | 否 | 60 日线定方向，MACD 定节奏，A/B/C/D 分级 | 趋势跟随、板块与个股同步转强时 |
| `tuige-shortline-trading` | 短线场景化交易框架 | 否 | 市场环境过滤 + 场景模块 + 统一 trigger/invalidation/risk | 短线题材轮动活跃、博弈性强的环境 |

## 已接入模拟/回测的交易策略

这些策略位于 `a-share-paper-trading` 的回测引擎中，已经可以直接做模拟或简单回测。

### `buy_and_hold`

- 定位：基准策略
- 逻辑：在回测开始时一次性买入，然后一直持有到结束
- 作用：不强调择时，主要用于衡量主动策略是否真的跑赢“买入并持有”
- 适合用途：做实验对照组、查看某只股票在给定区间的自然收益

### `sma_cross`

- 定位：经典均线交叉交易策略
- 逻辑：快线上穿慢线买入，快线下穿慢线卖出
- 参数：
  - `fast`：快线窗口
  - `slow`：慢线窗口
- 特点：
  - 规则简单，最容易参数化
  - 适合先搭实验框架，再做参数搜索
- 风格：
  - 更偏趋势跟随
  - 在单边或波段清晰时通常表现更稳定
  - 在震荡市容易被反复打脸

### `rsi_revert`

- 定位：均值回归交易策略
- 逻辑：RSI 低于买入阈值时买入，RSI 高于卖出阈值时卖出
- 参数：
  - `buy_rsi`
  - `sell_rsi`
- 特点：
  - 与均线交叉完全不同，更依赖价格回归
  - 适合和 `sma_cross` 做风格对比
- 风格：
  - 更适合震荡环境
  - 在强趋势行情里容易过早卖出或逆势抄底

## 选股与信号型策略

这些策略当前更偏“输出候选、信号、触发条件、风控框架”，不是已经统一接入回测引擎的成品策略。

### `a-share-strategy-mainboard-multi-swing-defensive`

- 定位：主板高流动性股票池中的趋势回踩决策系统
- 核心逻辑：
  - 从主板高流动性股票中筛出股票池
  - 用日线 `trend_pullback` 找潜在入场点
  - 输出上一交易日收盘信号和最新收盘信号两类买入参考
  - 对已有持仓输出卖出参考
- 附加过滤：
  - 成本过滤：估算往返交易成本后再筛
  - 鲁棒性过滤：用参数邻域一致性降低脆弱信号
- 输出：
  - 买入候选
  - 卖出候选
  - 现价快照
- 适用场景：
  - 盘前/盘后生成候选股
  - 与模拟盘配合，先出信号后人工执行
- 当前边界：
  - 不自动下单
  - 不做统一历史回测

### `macd-second-golden-cross`

- 定位：单票低位修复交易决策框架
- 核心逻辑：
  - 先确认前期下跌
  - 再区分第一脚、第二脚
  - 观察第二脚是否“变虚”，即动能衰减、背离增强
  - 用零轴下第二次金叉作为触发核心
- 输出：
  - 三档结论：观察、可试错、必须放弃
  - 入场条件、止损位、失效信号、仓位建议
- 适用场景：
  - 用户问“这是不是二次金叉，能不能做”
  - 低位修复、超跌反弹、筑底博弈
- 当前边界：
  - 更像单票判断模板
  - 不适合作为现成批量筛股器或统一回测器

### `macd-trend-resonance-stock-picker`

- 定位：趋势跟随型选股策略
- 核心逻辑：
  - 先用 60 日线和股价相对位置判断大方向
  - 再用 MACD 判断节奏、强弱和触发时机
  - 最终把候选分为 A/B/C/D 四档
- 规则重点：
  - 60 日线向上、股价站上 60 日线是硬条件
  - 日线 MACD 在 0 轴上方、金叉、红柱再放大属于加分项
  - 60 分钟级别共振可进一步强化信号
- 输出：
  - 候选池分级
  - 触发条件
  - 失效条件
  - 风险提示
- 适用场景：
  - 盘前趋势筛股
  - 盘中做回踩再上或突破确认
- 当前边界：
  - 是很成熟的规则型选股框架
  - 但还没有接入统一回测层

### `tuige-shortline-trading`

- 定位：短线场景化交易决策框架
- 核心逻辑：
  - 先判断市场环境是否支持短线交易
  - 再进入具体场景模块
  - 最后统一用退出规则和仓位纪律收束
- 场景模块包括：
  - 趋势回踩
  - 涨停后回调
  - 连板接力
  - 洗盘后突破
- 输出字段统一为：
  - `trigger`
  - `invalidation`
  - `risk`
  - `position_grade`
- 适用场景：
  - 短线交易者按场景做结构化决策
  - 对单票快速归类当前属于哪种可交易结构
- 当前边界：
  - 是一套短线规则框架
  - 不是已经落成统一参数化回测的模型

## 使用建议

如果目标是快速做实验，优先顺序建议如下：

1. `sma_cross`
   - 最容易参数化，适合作为 `experiments/scripts/` 的第一批实验对象
2. `rsi_revert`
   - 可与趋势策略做风格对照
3. `macd-trend-resonance-stock-picker`
   - 适合逐步转成批量筛股实验
4. `a-share-strategy-mainboard-multi-swing-defensive`
   - 已经非常接近可执行信号系统
5. `macd-second-golden-cross`
   - 更适合单票结构判断
6. `tuige-shortline-trading`
   - 规则丰富，但量化难度相对更高

## 一句话总结

- 现在已经能直接模拟/回测的，是 `buy_and_hold`、`sma_cross`、`rsi_revert`
- 现在已经有成熟规则、但还主要停留在信号/选股/决策层的，是其余 4 个策略体系

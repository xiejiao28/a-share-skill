# 交易策略参数调优实施计划

## 实施目标

完成第一阶段单票参数调优基础设施，围绕 `601669` 在近 3 年区间上验证：

- `sma_cross`
- `rsi_revert`

并保持：

- `buy_and_hold` 仅作为基准回测

## 实施步骤

### 1. 确认现有回测接口与结果字段

- 阅读 `a-share-paper-trading/scripts/paper_trading/engine.py`
- 确认 `run_backtest(...)` 的输入参数和输出结构
- 明确哪些字段可以直接复用，哪些指标需要在实验层补算

完成标准：

- 清楚列出 `sma_cross`、`rsi_revert` 当前支持的参数键

### 2. 设计实验脚本接口

在 `experiments/scripts/` 下增加实验脚本，建议最小接口支持：

- 指定 `strategy`
- 指定 `symbol`
- 指定 `start` / `end`
- 选择是否输出到文件

第一阶段默认值：

- `symbol=601669`
- `start/end=近3年`

完成标准：

- 脚本入口清晰，后续能重复运行

### 3. 实现 `sma_cross` 参数枚举

实现：

- 参数网格生成
- 第一版固定网格：
  - `fast`: `3, 5, 8, 10, 12, 15`
  - `slow`: `20, 30, 40, 60`
- `fast < slow` 约束
- 逐组调用回测
- 结果汇总

完成标准：

- 至少能输出一份 `sma_cross` 参数实验结果文件
- 首版结果覆盖预期 24 组参数组合

### 4. 实现 `rsi_revert` 参数枚举

实现：

- 参数网格生成
- `buy_rsi < sell_rsi` 约束
- 逐组调用回测
- 结果汇总

完成标准：

- 至少能输出一份 `rsi_revert` 参数实验结果文件

### 5. 统一收益/回撤平衡排序

为两个策略实验结果统一补齐派生指标：

- 总收益率
- 最大回撤
- 收益回撤比
- 交易次数

并给出默认排序结果。

完成标准：

- 能得到每个策略的参数排行榜

### 6. 结果落盘

把结果写入：

- `experiments/results/`

文件命名建议包含：

- 策略名
- 股票代码
- 时间区间
- 时间戳

完成标准：

- 结果文件结构稳定，可重复比对

## 验证步骤

完成实现后，至少执行：

1. 跑一轮 `sma_cross`
2. 跑一轮 `rsi_revert`
3. 检查输出文件是否落到 `experiments/results/`
4. 抽查排序前几名参数，确认约束成立

## 回滚点

如果实验脚本实现过程中发现回测引擎输出字段不足：

1. 暂停扩展实验脚本
2. 先补最小引擎输出
3. 重新执行实验层实现

如果实验脚本开始复制策略逻辑而不是调用引擎：

1. 回退编排层实现
2. 改回统一调用 `run_backtest(...)`

## 代码评审关注点

- 是否复用了现有回测引擎，而不是重复实现
- 参数约束是否明确
- 结果排序是否体现收益/回撤平衡
- 输出文件是否适合后续批量分析
- 是否把单票验证和多票泛化问题区分清楚

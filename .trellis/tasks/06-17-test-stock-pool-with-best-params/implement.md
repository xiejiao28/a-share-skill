# 股票池最优参数回测实施计划

## 实施目标

基于既有最优参数，完成指定 8 只股票池在近一年区间上的批量回测，并输出结果文件和简要对比报告。

## 步骤

### 1. 确认固定输入

- 固定股票池
- 固定近一年时间范围
- 固定两个策略参数：
  - `sma_cross 30/60`
  - `rsi_revert 25/60`

完成标准：

- 输入集合在脚本中清晰可见，且不需要重新调参

### 2. 在 `experiments/scripts/` 增加股票池批量脚本

脚本职责：

- 遍历股票池
- 对每只股票分别跑两个策略
- 统一收集结果

完成标准：

- 不复制策略逻辑
- 直接复用现有回测引擎或现有实验脚本骨架

### 3. 生成股票池级结果文件

输出到：

- `experiments/results/`

结果至少包含：

- 每只股票的两个策略结果
- 收益率
- 最大回撤
- 最终权益
- 交易次数

完成标准：

- 一个结果文件即可覆盖整组股票池

### 4. 生成简要报告

输出到：

- `experiments/reports/`

报告至少包含：

- 逐股票明细
- 按策略汇总
- 哪个策略整体更优的结论

完成标准：

- 报告简洁，结论优先

## 验证

- 成功跑完全部 8 只股票
- 每只股票都有两个策略结果
- `experiments/results/` 中有股票池级结果文件
- `experiments/reports/` 中有简要结论报告
- 抽查几只股票，确认结果字段完整

## 回滚点

- 如果批量脚本开始复制回测逻辑，立即回退到引擎复用方案
- 如果输出结构过于复杂，先收缩为最小 JSON 结果 + 简要 Markdown 报告

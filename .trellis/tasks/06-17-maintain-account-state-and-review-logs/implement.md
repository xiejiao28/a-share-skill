# 账户状态记录与手动复盘实施计划

## 实施目标

基于现有仓库框架，完成一套可持续使用的账户日志与手动复盘流程，包括目录结构、记录模板、复盘脚本或流程入口。

## 步骤

### 1. 建立日志目录结构

新增：

- `log/account/`
- `log/actions/`
- `log/reviews/`

完成标准：

- 目录存在
- 目录职责清晰

### 2. 定义三类记录模板

至少定义：

- 账户状态记录模板
- 操作记录模板
- 复盘记录模板

完成标准：

- 每类记录都能单独新增一个 Markdown 文件
- 文件名带时间标记

### 3. 实现账户状态写入 / 更新入口

支持：

- 用户输入账户信息
- 用户输入或更新持仓状态
- 写入 `log/account/`

完成标准：

- 至少能新增一条账户快照记录

### 4. 实现操作记录入口

支持用户在每次操作后记录：

- 做了什么
- 为什么这么做

并写入：

- `log/actions/`

完成标准：

- 至少能新增一条操作记录

### 5. 实现手动复盘入口

当用户说“开始复盘”时，流程至少要：

- 读取最近一次复盘
- 读取该时间之后的操作记录
- 读取最近一次账户状态
- 如可用则调用现有账户查询工具
- 输出新的复盘建议
- 写入新的 `log/reviews/`

完成标准：

- 至少能完成一次从读取历史到生成复盘记录的闭环

## 验证

- `log/` 子目录已建立
- 三类记录都能成功落盘
- 复盘时能找到最近一次复盘记录
- 复盘时能识别两次复盘之间新增的操作记录
- 复盘记录中包含建议

## 回滚点

- 如果脚本开始依赖复杂外部服务，回退到纯文件方案
- 如果记录结构过于复杂，先收缩为最小 Markdown 模板 + 手动触发流程

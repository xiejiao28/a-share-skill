# sma_cross 参数调优实施计划

## 实施目标

完成第一版 `sma_cross` 单票参数调优实验脚本，并把结果落到统一输出格式。

## 步骤

### 1. 确认 `sma_cross` 回测输入

- 阅读 `run_backtest(...)`
- 确认 `fast` / `slow` 参数键
- 确认返回结果里已有和缺失的指标

### 2. 定义参数网格

- 生成 `fast` / `slow` 参数组合
- 第一版固定：
  - `fast`: `3, 5, 8, 10, 12, 15`
  - `slow`: `20, 30, 40, 60`
- 应用 `fast < slow` 约束
- 预期有效候选共 24 组
- 同时保留可传自定义 `fast` / `slow` 候选列表的能力，供后续补充测试更长周期

### 3. 编排回测执行

- 对每组参数调用一次 `run_backtest(...)`
- 汇总原始结果

### 4. 补算统一排序指标

- 总收益率
- 最大回撤
- 收益回撤比
- 交易次数

### 5. 结果排序与落盘

- 按收益/回撤平衡排序
- 输出到 `experiments/results/`
- 使用父任务规定的统一字段

## 验证

- 至少成功跑出一份 `sma_cross` 结果文件
- 抽查前几组参数，确认 `fast < slow`
- 确认结果覆盖预期 24 组参数组合
- 确认文件名和字段契约满足父任务约束

## 回滚点

- 如果实验脚本开始复制策略实现，立即回退到引擎复用方案
- 如果结果契约不稳定，先收缩为 JSON 最小字段集

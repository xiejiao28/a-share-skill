# 优化 sma_cross 参数

## 目标

在现有 `a-share-paper-trading` 回测引擎之上，为 `sma_cross` 策略建立第一版单票参数调优实验，针对 `601669` 在近 3 年区间上搜索 `fast` / `slow` 参数组合，并按收益/回撤平衡输出结果排行。

## 已知信息

* 回测引擎已支持 `sma_cross`
* 当前可调参数为：
  * `fast`
  * `slow`
* 父任务已确定第一阶段只做单票验证：
  * 股票：`601669`
  * 时间区间：近 3 年
* 调优结果排序原则不是单看收益，而是收益 / 回撤平衡
* 第一版参数网格确认采用中等规模：
  * `fast`: `3, 5, 8, 10, 12, 15`
  * `slow`: `20, 30, 40, 60`
  * 并应用 `fast < slow` 约束
* 在首版跑通后，允许通过扩展 `fast` / `slow` 候选列表做补充验证，观察更长周期是否改善收益/回撤平衡

## 需求

* 在 `experiments/scripts/` 下增加 `sma_cross` 参数调优实验脚本或相关实现
* 枚举 `fast` / `slow` 参数组合
* 保证参数约束：
  * `fast < slow`
* 调用现有回测引擎，而不是重复实现策略逻辑
* 将结果写入 `experiments/results/`
* 第一版默认仅使用已确认的中等网格，不在首版扩大搜索规模
* 实验脚本应支持自定义 `fast` / `slow` 候选列表，便于补充测试更大的最大周期

## 验收标准

* [ ] 能对 `601669` 跑出一批 `sma_cross` 参数组合回测结果
* [ ] 输出结果包含收益、最大回撤、收益回撤比、交易次数等核心指标
* [ ] 排名体现收益/回撤平衡，而不是只看最高收益
* [ ] 结果文件成功落到 `experiments/results/`

## 明确不做

* 多股票批量验证
* 训练集/验证集拆分
* 图表化报告

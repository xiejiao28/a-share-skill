# log

`log/` 是个人账户状态、操作记录和手动复盘的工作区。

这个目录不是通用实验区，而是给 agent 和用户持续维护账户过程使用的。

目录约定：

- `log/account/`
  - 记录账户状态和持仓快照
  - 每次记录一个 Markdown 文件
- `log/actions/`
  - 记录每次操作和操作原因
  - 每次记录一个 Markdown 文件
- `log/reviews/`
  - 记录每次手动触发的复盘结果和建议
  - 每次记录一个 Markdown 文件

使用原则：

1. 每条记录单独成文件，文件名带时间戳
2. 用户手动触发“开始复盘”时，agent 需要：
   - 读取最近一次复盘
   - 读取两次复盘之间新增的操作记录
   - 读取最近一次账户状态
   - 结合仓库里的账户查询和分析流程生成新的建议
3. 每次复盘后必须写入新的 `log/reviews/` 记录
4. 每次用户操作后，如果用户提供了操作内容和原因，应该写入 `log/actions/`

如果后续 agent 处理账户维护、持仓更新、交易日志或复盘请求，优先从这个目录读取上下文，而不是只依赖对话历史。

# Exit and Failure Rules

## 核心问题

这一文档统一定义退出逻辑。  
各场景文档可以说明自己的 trigger，但退出统一回到这里复核，避免同一只票被多套卖点冲突解释。

## 退出分类

### 止盈兑现

适用于：

- 达到预设收益目标
- 冲高后出现兑现窗口
- 需要先收回一部分利润降低波动

输出建议：

- 是否部分兑现
- 是否保留底仓
- 若继续持有，后续观察什么

### 趋势失效

适用于：

- 有效跌破五日线或场景定义中的关键均线
- 跌破后无法快速收回
- 原本的趋势延续逻辑被破坏

输出建议：

- `invalidation: trend_broken`

### 结构失败

适用于：

- 突破失败
- 涨停后整理结构被破坏
- 接力逻辑中的强承接、强封板没有兑现

输出建议：

- `invalidation: structure_failed`

### 风险离场

适用于：

- 高位放量滞涨
- 高位高换手不涨
- 次日弱承接
- 放量下跌

输出建议：

- `exit_reason: risk_off`

## 共通信号

以下信号在多数场景下都应被视作高优先级观察点：

- 有效跌破五日线
- 高位放量滞涨
- 高位高换手不涨
- 封板不及预期
- 次日弱承接
- 放量下跌
- 结构突破失败

## 分场景补充

### 趋势类

重点看：

- 是否仍在关键均线之上
- 回调是否仍属缩量整理
- 是否出现明显转弱而收不回关键线

### 涨停后回调类

重点看：

- 是否跌破定义该结构的关键低点
- 是否由缩量整理转为放量转弱

### 接力类

重点看：

- 竞价承接是否明显不及预期
- 板块是否同步掉队
- 强封板逻辑是否落空

### 洗盘确认类

重点看：

- 确认突破是否失败
- 回到平台内后是否继续走弱

## 输出格式

建议统一输出：

- `exit_reason`
- `invalidation`
- 当前退出类型
- 触发退出的观察点
- 是否允许保留底仓

## 反例

以下做法应避免：

- 只讲买点，不预设失效条件
- 因为曾经强势，就忽略高位转弱
- 在接力逻辑失败后改口解释成波段持有

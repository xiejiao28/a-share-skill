# Stock Selection

## 核心问题

这一层只回答一个问题：哪些股票有资格进入观察池和候选池。  
它不直接定义买点，只负责先剔除明显不该碰的标的。

## 观察池标准

优先保留以下类型：

- 近期有涨停、突破或明显放量阳线
- 有成交量和流动性，不是长期无量的冷门票
- 所属题材有热度，或自身有辨识度
- 结构上至少没有明显破坏

## 候选池标准

在观察池基础上，再提高一级的条件：

- 个股比同板块更强，或具备板块带动性
- 量价关系健康，不是高位放量滞涨
- 位置和结构适合某一具体场景解释
- 与当前市场环境匹配

## 回避项

以下类型默认进入 `avoid_list`：

- 破位股
- 均线空头排列股
- 高位放量滞涨股
- 高位久盘股
- 无量冷门股
- 题材退潮后靠局部脉冲维持强势的个股

## 场景化筛选提示

### 趋势类优先看

- 五日线或关键均线之上运行
- 回调幅度可控
- 缩量整理后有再起预期

### 涨停后回调类优先看

- 近期首次或阶段性关键涨停后进入整理
- 回调中关键低点未被破坏
- 成交量缩而不散

### 接力类优先看

- 强题材
- 强卡位
- 强封板
- 强承接

### 洗盘末端确认类优先看

- 平台或箱体中已有整理结构
- 假跌破后有快速修复迹象
- 量能逐步压缩，等待确认突破

## 输出分层

建议统一输出为：

- `watchlist`
  - 有关注价值，但不能直接执行
- `candidate_pool`
  - 已经具备场景匹配资格，可进入具体买点判断
- `avoid_list`
  - 结构、量价或环境不匹配，明确回避

## 最低过滤原则

若以下问题任一回答为“否”，则不进入候选池：

1. 有流动性吗
2. 有结构吗
3. 有环境支持吗
4. 有场景归属吗

## 反例

以下做法应避免：

- 只因为价格低就纳入观察
- 只因为一根大阳线就直接进候选池
- 忽略题材强度与个股辨识度
- 把高位久盘误判成蓄势

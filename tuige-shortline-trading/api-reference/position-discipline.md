# Position and Discipline

## 核心问题

这一文档不负责判断买点，而是负责限制行为。  
规则的核心不是“如何重仓”，而是“何时轻仓、何时降频、何时不做”。

## 仓位等级

- `light`
  - 轻仓试错，适用于高风险场景或环境一般的场景
- `standard`
  - 标准执行仓位，适用于结构与环境都较清晰的场景
- `defensive`
  - 防守型仓位，偏观察或仅保留小仓位验证
- `no_trade`
  - 不开仓，只记录和观察

## 基本纪律

- 市场不支持时允许空仓
- 接力默认轻仓试错
- 趋势确认类可高于接力类，但不默认满仓
- 连续错误后降频
- 达到预期收益后可先兑现一部分
- 不满足触发条件时不强行出手

## 场景与仓位映射

### 趋势类

如果环境允许，且结构清晰、trigger 明确，可使用 `standard`。  
若环境一般或回踩仍未确认，则降为 `light`。

### 涨停后回调类

若缩量整理与关键低点都保持完好，可使用 `light` 到 `standard`。  
若仅有形态轮廓，没有确认动作，则保持观察。

### 接力类

默认 `light`。  
只有在市场环境、题材强度、承接质量同时满足时，才允许接近标准执行，但仍不以满仓为默认。

### 洗盘末端确认类

确认前不抬仓位。  
确认后再从 `light` 进入 `standard`，避免把猜底误当成确认。

## 降频规则

出现以下情况时，应主动降频：

- 连续多次 trigger 后都不能兑现
- 明显处于退潮环境
- 同类结构连续失效

降频动作包括：

- 降低仓位等级
- 只做低风险场景
- 暂停接力类交易

## 兑现规则

如果达到阶段预期，可优先考虑：

- 先兑现一部分
- 再根据结构是否延续决定是否保留底仓

这样做的目的不是追求卖在最高点，而是减少回撤和情绪反复。

## 禁止事项

- 用满仓弥补模糊判断
- 在 `no_trade` 环境下硬做接力
- 因为错过上一笔，就在下一笔盲目加仓
- 把纪律文档当成事后解释工具

## 输出格式

建议统一输出：

- `position_grade`
- 当前仓位建议
- 升级仓位的前提
- 降级仓位的前提
